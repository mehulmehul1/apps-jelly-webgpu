### 3.2 GPU Compute Optimization

#### Compute Shader Photocyte Updates
```wgsl
@compute @workgroup_size(256)
fn updatePhotocytes(
  @builtin(global_invocation_id) global_id: vec3<u32>
) {
  let photocyte_index = global_id.x;
  if (photocyte_index >= total_photocytes) { return; }

  // Parallel intensity calculation
  let pulse = sine_wave(time * frequency + phase);
  let decay = exp(-time_since_trigger * decay_rate);
  photocytes[photocyte_index].intensity = pulse * decay;
}
// 200,000 photocytes ÷ 256 threads = 781 workgroups
// Budget: 2ms for 200K photocytes
```

#### Temporal LOD System
```typescript
photocyteLOD: {
  near: { distance: 0-10, count: 2000, animation: 'full' },
  medium: { distance: 10-50, count: 500, animation: 'pulse_only' },
  far: { distance: 50+, count: 100, animation: 'steady_only' }
}
// Performance gain: 5-20× fewer updates for distant creatures
```

1. **Instancing:** 90% memory reduction for shared geometry
2. **Spatial Partitioning:** Octree-based culling (2-10× fewer updates)
3. **Emission Texture Baking:** Pre-bake static patterns (5-50× fewer calculations)
4. **Aggressive LOD:** 75% geometry reduction at distance
5. **Compute Shader Workgroups:** Optimal 256-thread groups
6. **Bioluminescence Budget Management:** Quality presets per tier
